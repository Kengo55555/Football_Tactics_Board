# Football Tactics Board（8人制 作戦ボード）

親子で一緒に選手を動かしたり、矢印でパスや走り込みを描いたりしながら、フットボールの作戦を視覚化するための Web アプリです。目的や AI 支援時の方針は [`claude.md`](./claude.md) を参照してください。

## 1. 必要な環境

次のソフトウェアがインストールされていることを想定しています。

| ソフトウェア | 推奨バージョンの目安 |
|--------------|----------------------|
| [Node.js](https://nodejs.org/) | 18 以上（LTS 推奨） |
| パッケージマネージャ | `npm`（Node に同梱） |

ターミナルで次のコマンドを実行し、バージョンが表示されれば準備完了です。

```bash
node -v
npm -v
```

## 2. 起動方法（開発サーバー）

### 2.1 初回のみ: 依存パッケージのインストール

プロジェクトのルートディレクトリ（`package.json` がある場所）で実行します。

```bash
cd "/Users/p211/Documents/個人/Football_Tactics_Board"
npm install
```

`node_modules` フォルダが作成され、React・Vite・Tailwind などが入ります。初回は数分かかることがあります。

### 2.2 開発サーバーの起動

```bash
npm run dev
```

実行が成功すると、ターミナルにローカル URL（通常は `http://localhost:5173`）が表示されます。ブラウザでその URL を開いてください。

- ファイルを編集するとホットリロードされ、保存内容がすぐ画面に反映されます。
- サーバーを止めるときは、ターミナルで `Ctrl + C` を押します。

### 2.3 アクセスできないとき

- ファイアウォールや VPN がローカルポートをブロックしていないか確認してください。
- 別のプロセスが 5173 を使っている場合、Vite が別ポートを自動選択することがあります。**ターミナルに表示された URL** を開いてください。

## 3. 本番用ビルドと確認

静的ファイルを `dist` に出力します。

```bash
npm run build
```

ビルド結果をローカルで確認するには:

```bash
npm run preview
```

表示された URL をブラウザで開き、開発時と同様に操作できるか確認します。本番サーバー（GitHub Pages、自前の Web サーバーなど）には `dist` 内のファイルを配置します。

## 4. 使用方法

### 4.1 画面の構成

- **左側（広いエリア）**: ピッチ（フィールド）。選手・ボール・矢印が載ります。
- **右側のパネル**: 「自チーム」「相手」のタブで選手一覧。ここから写真・名前・表示／非表示を操作します。
- **上部**: 「保存」「履歴」「リセット」ボタンがあります。

### 4.2 シナリオ（表示の切り替え）

ツールバー左付近のボタンで、説明したい場面に合わせて表示を変えられます。

| ボタン | 内容 |
|--------|------|
| **試合全体** | ピッチ全体と全選手（非表示にした選手を除く）を表示します。 |
| **守備シーン** | 自陣ハーフにフォーカスし、相手は一部のみ表示するプリセットになります。 |
| **攻撃シーン** | 敵陣ハーフにフォーカスし、攻撃参加メンバーを前方寄りにしたプリセットになります。 |

### 4.3 「動かす」ツール

1. **動かす** を選びます。
2. ピッチ上の**選手**または**ボール**をドラッグして動かします。
3. 右パネルで行をタップ／クリックすると、その選手が選択状態になります（写真アップロードの対象になります）。

### 4.4 「矢印」ツール（パス・走り込み）

1. **矢印** を選びます。
2. **パス**（黄色の実線）か **走り**（シアン色の点線）を選びます。
3. ピッチ上でドラッグすると、始点から終点への矢印が追加されます。短いドラッグは無視されます。
4. **矢印を消す** で、描いた矢印をすべて削除できます。

### 4.5 選手の名前・写真・表示

右パネルで各選手行の右端アイコンを使います。

| アイコン | 内容 |
|----------|------|
| 鉛筆 | 名前を編集（Enter で確定、Escape でキャンセル）。最大 12 文字。 |
| カメラ / アップロード | 写真を選ぶと、自動で小さめの正方形 JPEG として載ります。 |
| × | 写真を削除します。 |
| 目 | ピッチ上に出す／隠すを切り替えます。相手チームで「今説明にいらない人」を隠すのに便利です。 |

### 4.6 作戦の保存と履歴

1. 配置や矢印が決まったら、右上の **保存** を押します。
2. **名前**（必須、最大 40 文字）と **メモ**（任意、最大 140 文字）を入力し、**保存する** を押します。
3. **履歴** から過去の保存を一覧できます。カードの **読み込む** で盤面に復元、**ゴミ箱** でその保存を削除します（確認ダイアログあり）。

### 4.7 リセット

**リセット** は、選手位置・非表示・矢印・ズーム・ボールを**工場出荷の初期配置**に戻します。誤操作を防ぐため確認ダイアログが出ます。

### 4.8 キーボード

- モーダル表示中に **Escape** を押すと、モーダルを閉じられます。

## 5. データの保存場所と運用

### 5.1 ブラウザで `npm run dev` / `preview` を使う場合

保存機能は **`localStorage`** に書き込まれます（実装上は `ftb_storage:` というプレフィックス付きキーです）。

- **同一ブラウザ・同一オリジン**（同じドメイン・ポート）であれば、次回アクセス時も履歴が残ります。
- **別のブラウザ**や**シークレットウィンドウ**、**別 PC** では共有されません。
- ブラウザの「サイトデータを削除」「キャッシュ削除」で **保存した作戦が消える** ことがあります。大切な配置は、家族でスクリーンショットを取るなどのバックアップも検討してください。

### 5.2 `window.storage` を提供するホストに埋め込む場合

元のコンポーネントは **`window.storage`** という非同期 API（`list` / `get` / `set` / `delete`）を前提にしています。Electron やネイティブ WebView などで別実装に差し替える場合は、`src/storage-polyfill.js` を読み込まず、ホスト側で同じシグネチャを実装してください。

### 5.3 運用上の注意

| 項目 | 推奨 |
|------|------|
| 写真 | 枚数や解像度が大きいと保存 JSON が肥大化し、保存に失敗しやすくなります。必要な選手だけ載せる、古い履歴を削除する、などで容量を抑えてください。 |
| 端末の共有 | 家族で同じタブレットを使う場合は、複数のブラウザプロファイルを分けると、保存データも分かれて管理しやすくなります。 |
| アップデート | `package.json` の依存バージョンを変えたあとは、再度 `npm install` を実行してください。 |

## 6. トラブルシューティング

| 現象 | 確認すること |
|------|----------------|
| `npm install` が失敗する | Node のバージョン、ネットワーク、社内プロキシの設定。 |
| 画面は出るがスタイルが崩れる | `src/index.css` が読み込まれているか、`tailwind.config.js` の `content` が `src` を含んでいるか。 |
| 保存に失敗する | 写真を減らす・履歴を削除する。開発者ツールのコンソールにエラーが出ていないか。 |
| 履歴が空になる | 別 URL で開いていないか、ブラウザのデータ削除をしていないか。 |
