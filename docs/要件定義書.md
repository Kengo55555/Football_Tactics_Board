# 8人制フットボール作戦ボード — 要件定義書

## 1. 文書の目的

本書は、`src/tactics-board_1.jsx` に実装されている「8人制作戦ボード」アプリケーションに対する要件を整理し、開発・保守・受入判断の共通基準とする。

## 2. プロダクト概要

| 項目 | 内容 |
|------|------|
| 名称 | 8人制 作戦ボード（UI上はクラブ名表記あり） |
| 目的 | 親子が一緒に選手配置や矢印で「作戦」を視覚化し、保存して振り返れるようにする（詳細は `../claude.md`） |
| 形態 | Web ブラウザ上で動作するシングルページ型の React アプリ |

## 3. ステークホルダーと利用シナリオ

- **保護者・コーチ**: 守備・攻撃のシーン説明、保存データの管理。
- **子ども**: 選手のドラッグ移動、写真の割当、名前のカスタマイズへの参加。
- **前提**: タブレット／PC のモダンブラウザ。ポインター操作（マウス・タッチ）を想定。

## 4. 機能要件

### 4.1 ピッチ表示

- 縦長ピッチ（論理座標 幅 50 × 高さ 68）に、センターライン・サークル・ペナルティエリア・ゴール等のラインを SVG で描画する。
- 芝目のパターンで視認性を補う。

### 4.2 選手モデル

- 自チーム（ホーム）8名、相手チーム（アウェイ）8名の計16名を初期配置する（2-3-2 相当の位置関係）。
- 各選手は次の属性を持つ: `id`, `team`（home / away）, `x`, `y`, `num`（背番号）, `name`（表示名）, `photo`（Data URL または null）。

### 4.3 選手操作

- **移動**: 「動かす」ツール選択時、選手をドラッグして座標を更新する。
- **表示切替**: 右パネルから選手をボード上で非表示にできる（シナリオ説明用）。
- **名前編集**: 右パネルから編集。確定は Enter またはチェックボタン、キャンセルは Escape。
- **写真**: 画像ファイルを選択するとクライアント側で正方形リサイズ（既定 220px）し JPEG（品質約 0.82）の Data URL として保持する。削除も可能。

### 4.4 ボール

- ボールオブジェクトを1つ持ち、ドラッグで移動可能（「動かす」ツール時）。

### 4.5 ビュー（ズーム）

- **試合全体**: ピッチ全体を表示。
- **守備シーン**: 自陣ハーフのみ表示し、相手選手の一部を非表示・位置調整するプリセットを適用。
- **攻撃シーン**: 敵陣ハーフのみ表示し、自チームの一部を非表示・攻撃参加メンバーを前方に配置するプリセットを適用。

### 4.6 矢印（戦術線）

- 「矢印」ツール選択時、ピッチ上でドラッグして線分を描画する。
- 種別: **パス**（実線・黄色系）、**走り込み**（点線・シアン系）。
- 短すぎるドラッグは登録しない（しきい値あり）。
- 全矢印一括削除が可能。

### 4.7 保存・履歴

- 現在の盤面状態を名前・任意メモ付きで保存する。
- 保存内容: 選手配列、非表示 ID 一覧、矢印、ズームモード、ボール位置、タイムスタンプ等。
- 保存一覧を新しい順で表示し、読み込み・削除ができる。
- 読み込み時はモーダルを閉じ、トーストで通知する。

### 4.8 リセット

- 確認ダイアログのうえで、選手・非表示・矢印・ズーム・ボールを初期状態に戻す。

### 4.9 UI / UX

- ヘッダーにクラブロゴ（埋め込み画像）とタイトル、保存・履歴・リセット操作。
- 右サイドパネルで自チーム／相手のタブ切替と選手一覧。
- モーダル（保存入力、履歴一覧）。Escape で閉じる。
- トーストで成功・失敗・情報を短時間表示。
- 日本語フォントファミリを優先指定。レスポンシブ（`sm` / `lg` ブレークポイント利用）。

## 5. 非機能要件

### 5.1 パフォーマンス

- ドラッグ中は状態更新による再描画が追従できること（Pointer Events 利用）。

### 5.2 永続化

- 保存 API は `window.storage`（`list` / `get` / `set` / `delete`）を非同期で呼び出す。
- ブラウザ単体実行時は `localStorage` ベースのポリフィルで代替する（キーにプレフィックスを付与）。

### 5.3 容量・エラー

- 写真を多数保存すると JSON サイズが肥大化し保存に失敗する可能性がある。エラー時はユーザー向けメッセージを表示する。

### 5.4 ブラウザ対応

- ES モジュール、`async`/`await`、Pointer Events、`localStorage` を利用可能な環境を想定。

## 6. 対象外（現バージョン）

- サーバーへの同期、マルチユーザー編集。
- 動画再生、3D 表示。
- ルール変更に伴う人数可変（16名固定前提の UI）。
- 印刷レイアウト最適化（ブラウザの印刷機能に依存）。

## 7. 用語

| 用語 | 意味 |
|------|------|
| 論理座標 | ピッチ上の位置。表示領域（ビューポート）によりスクリーン座標に変換される。 |
| Data URL | `data:image/jpeg;base64,...` 形式の埋め込み画像文字列。 |
